# Denver MeshCore Naming Standard v1.1

## Part 1: Infrastructure

All names: 23 character max (hard limit)

#### Format: `[CITY]-[LANDMARK]-[TYPE]-[PUBKEY]`
- `CITY` ‚Äî Up to 7 chars
- `LANDMARK` ‚Äî Up to 7 chars (park, intersection, etc.)
- `TYPE` ‚Äî Node type (see below)
- `PUBKEY` ‚Äî The first two characters of the node's public key, for identification and uniqueness.

If landmark is prominent, `CITY` is replaced with `CO-`, and `LANDMARK` gets up to 12 characters.

##### Examples:
- `DENVER-CHSPARK-RC-9F` - Core repeater, Cheesman Park
- `DENVER-SLOAN-RE-9F` - Edge repeater, Sloan's Lake
- `LKWD-GRENMTN-RD-9F` - Dist repeater, Green Mountain
- `CO-REDROCKS-RC-9F` - Core repeater, Red Rocks (no city)
- `DENVER-RINO-T-9F` - Room server, RiNo

#### Node Types:
- `RC`: Core Repeater ‚Äî Backbone. Mountain/tower. Battery backup.
- `RD`: Distribution Rpt ‚Äî Bridges core to edge. Suburban elevated.
- `RE`: Edge Repeater ‚Äî Rooftop/residential. Mains power OK.
- `RM`: Mobile Repeater ‚Äî Vehicle or temporary.
- `T`: Room Server ‚Äî Fixed location.
- `TM`: Mobile Room ‚Äî Location changes.
- `TR`: Room + Repeat ‚Äî Room server w/ repeat on.

If a node also servers as an observer for [LetsMesh](https://analyzer.letsmesh.net/nodes), add `+O` at the end.

## Part 2: Companions

All names: 23 character max (hard limit)

#### Format: `[EMOJI] [HANDLE] [##]`

**NOTE:** The following is unchanged from v1.0.

- `EMOJI` ‚Äî One per person. Same on all your nodes. Claim in channel. First come, first served.
- `HANDLE` ‚Äî Your mesh alias. Not your real name. Nodes broadcast GPS ‚Äî real name + location is a bad combo.
- `##` ‚Äî Number by purpose starting at `01`, or use first 2 chars of pub key.

**Number by purpose, not hardware**. Swap devices, keep the name.

**DO NOT:**
- Use your real name
- Put hardware in the name
- Use different emojis per device ‚Äî one emoji per person
- Take someone else's emoji
- Go over 23 characters

- Deployed node names
- Claimed pub key prefixes

Check the list before deploying. Post in channel if adding something new.
